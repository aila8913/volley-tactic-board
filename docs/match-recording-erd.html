比賽記錄 ERD — 簡易版／進階版共用同一張 events 表

Match → Players / Sets → Rallies → Events 的巢狀關係維持原樣；今天討論的結論反映在 events 的欄位標記上——簡易版（賽中快速記分）點選目標（我方球員／對手(全體)／沒看到） → 選動作 → 選得失分，只填綠點與圈選欄位；進階版（賽後補填）才會用到藍點欄位。 「沒看到」這條路徑完全不建立 event，只直接記在 rallies 上。

今天的異動：fromX/fromY/toX/toY 要從現有的 .notNull() 改成可為 null， 簡易版才填得進這張表。

跟現有 lib/db/src/schema/ 的差異

  • matches / players / sets / rallies 維持現狀，沒有異動。
  • events.action 沿用現有 6 大類（serve / receive / set / attack / block / dig），簡易版跟進階版共用同一組分類，不再各自一套。
  • events.fromX / fromY / toX / toY 需要從 .notNull() 改成可為 null——支援 簡易版不記座標的情況。
  • 新增 events.side（home / away，必填）：對手自己失誤 導致我方得分時，這一球的「動作方」是對手，不是我方球員，需要獨立於 rallies.winner（得分方）之外記錄「誰做了這個動作」。
  • events.playerId 改成可為 null：點選「對手(全體)」時， 因為沒有存對手名單，playerId 留空，只靠新的 side = away 標示這是對方的動作。action 本身維持 .notNull()——「對手(全體)」一樣要選動作大類。
  • 「沒看到」路徑（連是哪一方動作都判斷不出來）不建立 event，直接把 得失分記在 rallies.homeScore / awayScore / winner 上，event 表不用因此 再新增任何 nullable 欄位。
  • tactics 表（戰術板存檔用）跟這條記錄／統計的資料流無關，此圖刻意不畫進來。
