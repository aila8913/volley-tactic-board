# 設計規範（Design Spec）

這份文件是專案的「設計字典」。填好之後，往後請 Claude 做 UI / 畫面時，
只要說「照設計規範來做」，就會照這裡的規則走，不用每次重新丟截圖或重講一次。

> 這份規範是綜合兩張參考圖定案的：**版面/留白/玻璃質感**取自網球儀表板參考圖，
> **配色（暗底 + 螢光綠）跟文字風格**取自運動投注儀表板參考圖。細節見各節說明。

---

## 1. 品牌基調

- **一句話形容這個 app 的感覺**：
  深色、俐落、帶一點科技感的運動數據儀表板 —— 像賽事直播疊加畫面，
  但版面乾淨、卡片之間留白充足，局部用「毛玻璃」卡片增加層次感。
- **參考對象**：
  - 版面/留白/玻璃質感 → 網球儀表板參考圖（乾淨網格、圓角卡片、照片上疊毛玻璃文字牌）
  - 配色/文字風格 → 運動投注儀表板參考圖（深黑背景、螢光萊姆綠強調色、粗體白字）

## 1a. 品牌標誌（Logo）

**2026-07-18 定案**（mockup：https://claude.ai/code/artifact/3b17a651-61d3-43c5-a3f2-36f1e334071c）。

- **概念：旋轉中的排球**。三道粗弧線以 120° 旋轉對稱排成三叉旋渦——排球皮瓣接縫的
  抽象化，也是球體高速旋轉時的殘影，單一造型同時講完「排球」跟「速度」，不需要任何
  輔助線條。
- **構造**：`viewBox 0 0 100 100`，三道弧線 `M 88 50 A 27 27 0 0 0 34 50`
  （`stroke-width 6.5`，`stroke-linecap round`），分別 `rotate(0/120/240 50 50)`。
  外圈是**三段弧＋圓形端點**的環（`circle r=44`，`stroke-dasharray "74.15 18"`，
  同樣 `round` 收尾）——**不是實心圓**，破口呼應旋渦的三重旋轉節奏，線條收尾統一
  才不會有「手繪弧線＋死板幾何圓框」兩種語彙硬湊在一起的問題。
- **微光**：外環用「模糊墊底＋清晰線在上」疊出微微發光（`feGaussianBlur
stdDeviation 1.5`），呼應整體發光語言，但**極克制**——不是霓虹感。
  **favicon／24px 以下尺寸不發光**，極小尺寸模糊會糊成一團，寧可犧牲光暈保銳利。
- **配色**：主要版＝深底＋萊姆綠弧線＋米白半透明環（`public/favicon.svg` 用的版本）；
  另有玻璃底單色版、萊姆綠底反白版、玻璃質感版（後者只用於 hero／大尺寸展示，UI
  內一律用主要版，玻璃質感縮小後高光會糊掉）。
- **字標**：品牌名稱字體用 `Sixtyfour`（見第 3 節），兩行排列（例：`VOLLEY` /
  `BOARD`，`BOARD` 用萊姆綠），不加斜切（Sixtyfour 本身已經很有個性，斜切會讓
  數位方塊字看起來歪斜不穩）。**目前品牌名稱本身尚未定案**，字標文字待確認。
- **落地狀態**：mark 已經是正式 `public/favicon.svg`；戰術板背景的 `.tb-mark`
  字標用同一套邏輯但還是佔位文字，兩者品牌名稱定案後要一起更新。

## 3. 字體（Typography）

- **主要字體（標題/內文）**：`Space Grotesk`（Google Fonts）—— 幾何感、粗體時很像參考圖裡
  "Dashboard" "224 Wins" 那種運動數據標題感。中文字則用 `Noto Sans TC`。
- **等寬字體（比分、計時器、數據）**：`JetBrains Mono` —— 用在比分、球員背號、時間這類需要
  對齊、有「數據感」的地方（呼應運動投注圖裡的比分數字風格）。
- **背景字標字體（Sixtyfour）**：`Sixtyfour`（Google Fonts，電路板/數位顯示風格）——2026-07-17
  戰術板材質強化（issue #134 Track B）新增，**只用在背景的置中大字標**（見第 5 節
  「背景字標」），不是內文/標題字體，不要拿去套一般文字。跟 Anton 的教訓相反：這款字體
  就是設計來當大字展示用的視覺量體，不是給人一個字一個字讀的。
- **徽章字體（Anton）：PR #129 review 後決定不用**——原本想拿來做隊名/大標題，但 Anton 只有
  一個字重（400），套在 17–18px 的卡片標題正好是它最難讀的尺寸，而且中文完全不會 fallback
  成它、只會顯示系統字體，等於白套。比賽列表頁的標題/隊名一律用 `Space Grotesk`（跟內文同一套），
  Anton 只留給真的需要大字展示、而且內容以英文/數字為主的場合才考慮用。
- **字級規則**：

| 用途                                    | 大小                                      | 粗細                       |
| --------------------------------------- | ----------------------------------------- | -------------------------- |
| H1 標題（頁面主標，如 "Dashboard"）     | 28–32px                                   | Bold (700)                 |
| H2 標題（區塊標題，如 "My Next Match"） | 18–20px                                   | Semibold (600)             |
| 內文 Body                               | 14px                                      | Regular (400)              |
| 小字/註解 Caption（時間戳記、標籤）     | 11–12px，字距加寬 (letter-spacing 0.05em) | Medium (500)，常搭配全大寫 |

## 4. 間距與版面（Spacing & Layout）

- **基礎間距單位**：4px 為一單位（Tailwind 預設的 spacing scale 直接套用即可，不用另外定義）。
- **卡片圓角 Border Radius**：大卡片/容器 `rounded-2xl`（16px）；小卡片、按鈕、標籤 `rounded-full`
  或 `rounded-xl`（12px），呼應兩張參考圖偏圓潤的卡片邊角。
- **陰影風格 Shadow**：輕微。深色底幾乎不用陰影撐層次，改用「玻璃卡片」本身的邊框
  （`border border-white/10`）+ 輕微內陰影或外陰影 `shadow-lg shadow-black/20` 來區分層級，
  避免像淺色卡片那樣需要重陰影。
- **玻璃質感做法（Glassmorphism）**：卡片背景用 `bg-white/5` 或 `bg-white/10`，
  加 `backdrop-blur-md`，邊框 `border border-white/10`。用在疊在照片/影片上的資訊牌、
  或需要分層但不想太厚重的卡片（例如戰術板上的球員資訊卡）。
  **注意（PR #129 review 抓到的坑）**：`backdrop-blur` 模糊的是元件背後的內容——如果背景是
  純色，模糊完還是同一個純色，玻璃感不會真的出現，只是白付合成層的效能成本。套用在純色背景的
  頁面（例如比賽列表）時，要記得在背景加點材質（淡淡的斜線/網格紋理、mesh gradient、或一團
  柔和的強調色光暈）讓 blur 有東西可以模糊。
  **實作決定（戰術板材質強化，2026-07-17）**：內層小卡片（球員列、已儲存戰術）的玻璃感
  再加深一階——`bg-white/[0.08]` → `bg-white/[0.11]`、`backdrop-blur-md` → `backdrop-blur-lg`、
  邊框 `border-white/[0.14]` → `border-white/[0.18]`，讓它跟外層 chrome（header／側欄，維持
  原本較淡的 `bg-white/[0.02]` `backdrop-blur-sm`）對比更明顯，讀起來更扎實、less see-through。
  這是照夥伴反饋「毛玻璃太多有點不像戰術規劃該有的沉穩感」調整的方向：**不是拿掉玻璃，是讓
  玻璃更不透明、更安靜**。
  **原則（同一次調整定案）：發光效果只留給真正需要被看見的狀態，不要做成常駐的裝飾**。
  球員列的 hover 發光（見 `index.css` `.roster-row:hover`）是唯一保留的發光——只在滑鼠移過去
  這種有意義的互動狀態才出現。背景裝飾性的光束/光暈**不要**做成飽和度高、常駐可見的效果
  （之前一版光束用螢光綠網點紋理常駐發光，跟這條原則衝突，已收斂成幾乎察覺不到的斜向光影，
  見第 5 節「背景字標」）。
- **版面寬度**：桌機優先（這是戰術板工具，主要在筆電/平板上用），最大內容寬度約
  `1440px`，兩側留白。手機版先求「看得到比分跟戰術板」，複雜編輯功能可以之後再優化。

## 5. 元件風格（Components）

> 基準是 shadcn/ui 預設元件，以下是要調整的地方。

- **按鈕 Button**：主要按鈕 = 萊姆綠底、深色文字（`bg-[#C6F135] text-[#121310]`），
  `rounded-full`，無陰影或極輕陰影，hover 時稍微加亮。次要按鈕 = 透明底 + 白色半透明邊框
  （`border border-white/20 text-white`），像參考圖裡的膠囊型按鈕。
- **輸入框 Input**：深色玻璃質感（`bg-white/5 border-white/10`），focus 時邊框變萊姆綠。
- **卡片 Card**：預設用玻璃卡片風格（見第4節），文字卡片可用米白底 + 深色文字做對比
  （呼應網球儀表板圖中偶爾出現的淺色卡片，用來突顯重點數據）。
- **導覽列 Navigation**：仿網球儀表板的左側直式導覽列 —— 深黑底、圖示 + 文字、
  active 項目左側有一條萊姆綠指示條，icon 也變萊姆綠。
- **戰術板畫布 Tactics Board Canvas**：球場底色用暖木色或橄欖綠，
  己方球員標記用萊姆綠圓點，對方用珊瑚紅圓點，移動軌跡線用米白半透明虛線，
  選取中的球員可加一圈萊姆綠光暈（呼應整體強調色）。
  **實作決定（2026-07-15，改版）**：暖木地板色 `#C9A25D` 試過之後覺得不符合整體深色調性、
  跟現實球場配色的記憶點也對不上，換成單一深青漸層（`court-gradient`：`#12403f` →
  `#1c5654` → `#2a6e6a`，由對手端到我方端）。冷色調對己方萊姆綠跟對方珊瑚紅兩種球員點
  都是安全的對比組合，不會有球場底色跟球員點顏色太接近、看不清楚的問題（這是選色時特別
  排除掉的方向，例如暖色球場配珊瑚紅球員點）。球場外框線、球網/攻擊線、「對手」「我方」
  文字都改成米白 `#F5F5F0` 半透明 60%（`stroke-opacity`/文字直接用這個色，原本是黑色
  `#111`，在深色球場上會看不到）。球員標記本身的敵我配色邏輯
  （`PlayerNode.tsx` / `Markers.tsx` / `Court.tsx`）維持不動，這次沒有動任何判斷邏輯，
  只調了球場底色跟中性線條的顏色。

- **戰術板的「手繪白板感」已退役**：戰術板原本有一套獨立於這份規範的視覺語言——手寫體
  （`--font-display`，Permanent Marker）＋ `wobbly-border`（不規則圓角模擬手繪邊框）＋
  `wobbly-svg`（SVG 亂數位移濾鏡，讓球場線條看起來手繪）＋漫畫式硬邊陰影
  （`shadow-[2px_2px_0_0_#111]`）。跟這份規範的玻璃卡片語言放在一起非常突兀（純白卡片
  浮在深色底上），2026-07-15 決定整個戰術板頁面（`TacticsBoard.tsx` 跟它底下的
  `RotationTable.tsx` / `TacticsBoardPanel.tsx` / `RotationSwitcher.tsx`）全部退役這套
  風格，統一改用本文件的玻璃卡片 + Space Grotesk。
  **2026-07-22 更新**：計分表中間計分區（`ScoreSheet.tsx`／`ScoreSheetCourt.tsx`／
  `RadialMenu.tsx`，issue #131）也套用完這份規範了——右欄站位／統計面板先前已由 #120 換成
  深色玻璃，這次補齊的是中間白底的計分/球場區。`RadialMenu.tsx` 是最後一個還在用
  `wobbly-border` 的地方，換成跟這份規範一致的玻璃圓角按鈕後，`wobbly-border` 這個 CSS
  class 已經沒有任何消費者，直接從 `index.css` 刪掉了。全站到這裡已經沒有手繪風的殘留元件；
  `--font-display`／Permanent Marker／Caveat 這幾個字體變數跟 `@import` 也已經確認沒有任何
  `.tsx` 在用，一併清掉。

- **背景字標（issue #134 Track B，2026-07-17 定案）**：戰術板背景加了兩層裝飾材質，都寫成
  plain CSS class（`.tb-beam`／`.tb-mark`，同 `.roster-row:hover` 的理由：Tailwind 任意值
  解析多層 rgba 容易出錯，見 `index.css` 註解），不是用 Tailwind 任意值堆出來的：
  - **`.tb-mark`**：置中放大的品牌字標，疊兩層文字（`.layer-glow` 模糊當暈、`.layer-core`
    清晰保留可讀性），字體用 `Sixtyfour`（見第 3 節）。尺寸故意比球場卡片寬，讓它從球場
    兩側露出來，而不是完全被不透明的球場卡片蓋住。目前文字是 `VOLLEY` / `BOARD`
    佔位——**品牌名稱還沒定案**，之後兩人想好名字後直接改文字內容即可，樣式不用動。
  - **`.tb-beam`**：極淡的斜向光影，`filter: blur(30px)`、峰值不透明度只有 0.07，用來破
    「整片平的」單調感，但刻意做得幾乎察覺不到——見上面「發光只留給真正需要被看見的狀態」
    原則，這不是常駐發光的東西，只是材質上的細微變化。
  - 兩者都是 `position: absolute`，所在頁面（`TacticsBoard.tsx`）的外層 wrapper 因此需要
    `position: relative`，前景 UI（header、三欄主體）也要有自己的 stacking context
    （`relative z-10`）疊在上面，不然背景層會蓋到前景 UI。

## 6. 其他偏好

- **要避免的風格**：太多層陰影堆疊、彩虹漸層、大面積純白背景（整體以深色為主調）。
- **Icon 風格**：線條 outline 風格為主（呼應兩張參考圖的細線 icon），統一用
  `lucide-react`（專案已可用）。強調狀態時 icon 可填色（filled）並轉成萊姆綠。
- **動效 Motion**：輕量即可 —— 卡片 hover 輕微上浮 + 陰影加深、按鈕 hover 輕微變亮、
  數字變化可以有簡單的 count-up 動畫。避免太浮誇的轉場，畢竟是要拿來記錄比賽用的工具，
  操作流暢度優先於視覺效果。
  **規範放寬（PO 拍板，2026-07-18，issue #134）**：常駐環境動效（不是使用者互動觸發、
  一直在畫面上跑的那種）原本完全排除在「輕量」之外，現在放寬成——**每頁最多一個環境級
  動效，必須慢（10 秒以上一輪）且低亮度，不能搶操作注意力**。戰術板背景的邊緣線條光
  （`.tb-mark`／logo 外環發光，12 秒繞行一圈）是這條規則的範本案例。互動觸發的動效
  （hover、count-up）不受這條數量限制，維持原本「輕量即可」的原則。

## 使用方式

跟 Claude 說：
