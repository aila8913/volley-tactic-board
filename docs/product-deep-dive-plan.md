# 產品深度思考：執行計畫（2026-07-08）

> 這是 5 項產品面深度工作的執行步驟拆解，每項都設計成**可獨立一個 session 完成**。
> 背景脈絡見 `docs/product-vision.md` 與 issues #73–#77。
> 依賴關係：T1 → T2、T4；T3、T5 完全獨立可隨時插隊。
> 模型建議：T1/T3 推理最重（值得用最強模型）；T5 是查資料，輕量模型即可。

---

## T1. 統計反推對照表（#73 事件文法領域模型）★最優先、最不可逆

**產出**：新文件 `docs/event-grammar-spec.md` ＋ 摘要貼到 #73。

1. **窮舉統計清單**：讀 `docs/product-vision.md`、`docs/product-spec.md` 功能三、
   `docs/marketing-page-draft.md` 的 features 區、記憶中 SpikeVision BasicStatsPage
   的參考項目（動作分布、評價分布、sideout%、球員表現、比分趨勢、球員統計表），
   列出「產品承諾過或想要的每一個統計數字」。
2. **逐條寫推導式**：每個統計寫成 `統計 = f(events/rallies/sets 的哪些欄位)` 的
   純函數描述，對照現行 schema（`lib/db/src/schema/index.ts`）。
3. **標出缺口**：推不出來的統計 = schema 表達力缺口。已知懸案要在此定案：
   換人（#42）、暫停（#44）、犯規/Outcome 細節（#51）到底進 events 還是專用表。
4. **產出對照表**：三欄「統計 ⇐ 推導式 ⇐ 所需欄位/缺口」，缺口整理成 schema
   決策清單（每條標：可後補 vs 不可逆）。
5. 摘要貼 #73，需要使用者拍板的決策條列在最上面。

## T2. 記錄成本預算（#74）——依賴 T1

**產出**：量化模型寫進 #74 comment（或 `docs/recording-cost-budget.md`）。

1. **定基準節奏**：業餘比賽每球間隔估 15–30 秒（若使用者有比賽影片可實測幾個
   rally 校正；沒有就用區間跑敏感度分析）。
2. **枚舉每種記錄流程的操作成本**：現行簡易版（點人→選動作→勝負）、
   「對手/沒看到」快速鍵、#21 的兩點座標流程、#51 的子分類流程——各算
   點擊數＋視線離開球場的秒數。
3. **建預算表**：單人記錄下，每球間隔內塞得進哪些流程？超過預算的組合標紅。
4. **畫出簡易/進階硬分界**：賽中來不及的全部歸進階版（賽後影片補填）。
   結論直接回答 #50（情境限制值不值得加點擊）、#51、#21 的分層歸屬。

## T3. 離線可靠性契約（#75）——獨立，擋部署 #26

**產出**：新文件 `docs/offline-reliability-contract.md` ＋ 實作任務清單貼 #75。

1. **窮舉失敗場景**：rally 記到一半斷線、undo 與佇列中寫入交錯、佇列未清空就
   reload、同帳號兩台裝置、#63 的未開球空局、#64 的背景寫入失敗。
2. **定義契約**：什麼叫「已記錄」（按下當下即算）、client 端產生 UUID 的冪等重送、
   佇列順序與重播語意、失敗多久後要讓使用者知道、reconcile 的 UI 形態。
3. **寫成可驗證規格**：每條契約配一個「怎麼測」（斷網手測步驟或單元測試）。
4. **拆實作任務**：IndexedDB 佇列、API 冪等 key、`useScoreSheetController` 改造，
   各開成 issue 或貼回 #75 當 checklist。

## T4. 球線分布圖的分析語意（餵 #65/#21/#76）——依賴 T1

**產出**：分析頁資訊設計 spec（可併入 T1 文件或獨立 `docs/analytics-semantics.md`）。

1. **列教練問題**：佔位怎麼調、對手主攻打哪、我方誰接不穩、發球瞄誰……
   （可請使用者和原型 TA 補真實問題）。
2. **每個問題配一張圖**：需要的圖型＋過濾維度（按對手輪次？按攻擊發起位置？）
   ＋回答此問題所需的最小資料欄位。
3. **樣本數誠實機制**：定義每張圖的最低樣本門檻，不足時 UI 怎麼表達（灰掉/
   顯示 n=？），避免 20 球的熱區被當結論。
4. **回灌三處**：所需欄位 → T1 對照表；記錄端要記什麼 → #21；
   「分析結果→回頭修正排陣」的入口 → #76。

**產出**：直接更新 `docs/marketing-page-draft.md`，把【未驗證】換成有出處的事實。

   驗證 0 區價格帶圖，附查證日期與連結。
2. **商標檢索**：TIPO（台灣智慧財產局）檢索「球跡」、WIPO/一般搜尋「BallTrace」
   的名稱衝突；只做初篩，結論標「初步檢索，非法律意見」。

   手續費，驗證「每隊每月 NT$100–200」在成本上站不站得住。
4. 每處修改保留原【未驗證】標記的替換紀錄（出處＋日期）。
